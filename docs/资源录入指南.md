# L-2 项目 · 资源录入指南

**用途**:供学习资源（视频/文章/习题/笔记）录入同学使用,与题库录入流程并行,字段更精简。
**模板**:[`data/resources_template.xlsx`](../data/resources_template.xlsx)
**版本**:2026-05-22

---

## 一、录入流程

```
            ┌─ 录入资源(填 xlsx) ──┐
你          │                      │
            └─ 跑导入命令 ─────────┘
                     │
                     ▼
            resources 表入库
                     │
                     ▼
            跑 init_rag(模型 + 索引) ──► Chroma 向量库
                                              │
                                              ▼
                                   plan Agent 推荐路径时
                                   会拿候选资源喂给 LLM
```

## 二、xlsx 字段对照

| 字段 | 必填 | 说明 |
|---|---|---|
| `subject` | 是 | 学科,目前只有 `管综数学` |
| `category` | 是 | 知识模块,见 [`概念清单.md`](概念清单.md);例如 `代数` |
| `knowledge_point` | 是 | 具体知识点,见概念清单;例如 `等差数列` |
| `title` | **是** | 资源标题,会直接作为路径推荐项展示给学生 |
| `type` | 是 | 类型:`video` / `article` / `exercise` / `note` |
| `url` | 否 | 资源链接,可留空 |
| `estimated_minutes` | 是 | 预计学习分钟,正整数 |
| `summary` | **是** | 摘要,**RAG 检索的关键字段**——直接决定推荐准不准。**不要复制 title**,要写资源真实涵盖的知识要点 |
| `source` | 否 | 来源,如 `B站-张同学讲管综` / `《数学历年真题》第 5 章` |
| `tags` | 否 | 标签,逗号分隔 |

> **summary 写得好不好,直接决定 RAG 命中率**。最好包含:
> - 资源讲了哪些子知识点
> - 用什么方法/技巧讲解
> - 学完后能解决哪类题

## 三、命令清单

```bash
cd ~/workspace/learning-path

# 1. 把同学的 xlsx 导入 resources 表
.venv/bin/python -m scripts.import_resources <xlsx 路径>

# 2. 重建向量索引(首次会下载 bge-m3 ~2.3 GB,后续秒级)
.venv/bin/python -m scripts.init_rag
```

import 失败的行会列在终端输出里,按提示改 xlsx 后再跑就行(脚本幂等,按 (title, url) 去重)。

## 四、与题库的关系

- 题库(`questions` 表)——学生答题,产出 `mastery`
- 资源(`resources` 表)——LLM 规划路径时从中挑推荐项

两者通过 **同一套 33 个 concept_code 锚定**,所以 (category, knowledge_point) **必须严格按 [概念清单](概念清单.md) 来填**,否则 alias 反查失败。

## 五、常见问答

**Q: title 该写多详细?**
A: 让学生看到能立刻判断是否要点开。比如 `等差数列通项 + 求和速记法`比`数列视频 1`好。

**Q: 一条资源涵盖多个知识点怎么办?**
A: 选**讲得最充分**的那一个 concept 填,在 summary 里把另外几个也提一笔(便于 RAG 检索时也能命中)。

**Q: 我没找到合适的链接,只是想录条占位?**
A: url 留空、summary 写真实摘要即可,demo 阶段已经够用。

**Q: 录完一批,系统就能推荐了吗?**
A: 不,**导入数据库后必须再跑一次 `init_rag`** 才会更新向量索引。每次 xlsx 导入后跑一次即可。
